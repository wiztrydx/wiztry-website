# WizTry公式サイト 実装仕様書

このリポジトリはAstro 5の静的サイト。実装エージェントは担当ページのみを作成すること。

## 共通ルール（厳守）

1. **事実の捏造禁止**。本仕様書に書かれていない数字・社名・実績・料金を勝手に作らない。文言は本仕様書のコピーをそのまま使う（てにをはの微調整は可）。
2. 「No.1」「日本一」「熊本一」等の最上級表現は**使用禁止**（景表法対応）。
3. レイアウト・装飾は `src/styles/global.css` の既存クラスを最大限使う。ページ固有のスタイルは `<style>` ブロックで最小限追加してよい（既存のトーンを守る。色はCSS変数を使う）。
4. 共通部品: `src/layouts/Base.astro`（Props: title, description, onDark, ogImage）、`src/components/SectionHead.astro`（Props: en, title, lead, center）、`CtaBand.astro`、`LogoWall.astro`（mode: 'grid'|'marquee'）、`Stats.astro`。共通データ: `src/data/site.ts`（SITE, STATS, CLIENTS, INDUSTRIES, MEDIA, NAV）。
5. 各セクションのラッパは `<section class="section">`＋`<div class="container">`。交互に `section--alt`（薄グレー）や `section--navy` を使いリズムを作る。
6. 要素の出現アニメは `class="reveal"`（遅延は `reveal-d1`〜`d4`）。付けすぎない（セクション見出し・カード単位）。
7. 画像は `loading="lazy"`（ファーストビューを除く）。altは日本語で意味のある文言。
8. 下層ページは冒頭に `.page-hero`（en見出し＋jpラベル＋desc）を置く。Base呼び出しは `onDark={true}`。
9. ページ末尾（フッター直前）に `<CtaBand />` を置く（contact/thanks/privacy/404を除く）。
10. TypeScriptエラーを出さない。`npm run build` が通ること。

## 使える写真（public/photos/）

- `hero-seminar.webp` … 大会場セミナーで登壇する代表と満席の聴講者（トップヒーロー用）
